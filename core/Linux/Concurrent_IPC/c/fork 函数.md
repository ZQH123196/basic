
## 程序运行简述

1）在内存中划出一片内存空间。
2）将硬盘上可执行文件的机器指令拷贝到划出的内存中。
3）pc 指向第一条指令，cpu 取指令运行。

当有 OS 时，以上过程都由 CPU 一手包办。

在 Linux 中，OS 提供了两个 API：fork()、exec().

fork：开辟一块新的内存空间，并将运行该函数进程的内存空间原样拷贝过去。

exec：将程序代码（机器指令）拷贝到开辟的内存空间中，并让 pc 指向第一条指令，CPU 执行。运行起来的进程会与其他的进程切换着并发运行。

## fork 函数

功能：
从调用该函数的进程复制出子进程，被复制是作为父进程，复制出来的作为子进程。

复制：依照父进程原样拷贝其虚拟内存空间，开辟一块新的虚拟内存空间来存放。

开辟一块新的空间，可以让 exec 加载子进程的代码和数据而不影响父进程，保证了进程间的内存空间独立。

返回值：
1）父进程的 fork，成功返回子进程的 PID，失败返回 -1，errno 被设置。
2）子进程的 fork，成功返回 0，失败返回 -1，errno 被设置。
> 根据父子进程运行 fork 不同的返回值，可以区分父子进程去做不同的事情。
>> 当然，这样父子进程运行的代码全部都堆到一起，就会很难 debug，也因此很掉头发。

注意：**子进程是原样拷贝的父进程的内存空间**，因此在 fork 时当前代码取指指向哪里也会被拷贝，所以子进程会从 fork 哪里开始运行而不是从头。也要注意变量和缓存区（特别是 printf 缓冲区）这些也都拷贝过去了。
但是文件描述符并不属于拷贝范畴，或者说 fd 与代码缓存这些不存放于同一个地方，fd 属于 OS 管理。



## 子进程会继承的与不会继承的

子进程将会继承以下属性：
* 环境 Environment
* 堆栈
* 用户 ID， 用户组 ID
* 进程组 ID
* 会话期 ID
* 控制终端
* 当前工作目录
* 根目录(/)
* 文件创建时的屏蔽字
* 环境变量
* 已经打开的文件描述符
* 共享内存
* 执行时关闭（Close-on-exec）标志
* 信号（Signal）控制设定 

子进程独立的属性：
* 进程 ID
* 父进程设置的锁不被继承
* 子进程不继承父进程的进程正文（text），数据和其他锁定内存（memory locks）
* 不继承异步输入和输出
